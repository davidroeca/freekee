# Round-trip fixture generation guide

This document is the developer's recipe for creating `tests/roundtrip/fixtures/`. Once generated, these files are checked into the repo and serve as the ground truth for KDBX compatibility.

## Conventions for all fixtures

- **Test password**: `correct horse battery staple` — used for every fixture except where noted. Public xkcd reference; never use a real credential.
- **KDBX version**: 4.1 (KeePassXC default in 2.7+) unless the fixture name says otherwise.
- **Cipher**: ChaCha20 (KeePassXC current default) unless noted.
- **KDF**: Argon2id with KeePassXC defaults (memory `~64 MiB`, iterations `~10`, parallelism `~2`. Whatever your KeePassXC suggests is fine; the fixtures lock in *whatever* parameters were used, not specific values).
- **Database name**: matches the fixture directory name.
- **`password.txt`**: contains the password followed by a single trailing newline. No quotes, no extra whitespace.
- **`expected.json`**: generated by `cargo run -p kdbx --bin dump-expected -- db.kdbx` once that tool exists. The `expected.json` files are produced later by the dump tool and committed in a follow-up. In that same follow-up, this line should be updated to only include the first sentence.

When KeePassXC asks about KDF parameters during database creation, accept the defaults. Its "Benchmark 1 second delay" button is fine. Do not manually weaken parameters; the audit crate's bad-config fixtures are generated in-memory, not stored here.

## Directory structure to create

```
tests/
└── roundtrip/
    ├── README.md                          # short pointer to this doc
    └── fixtures/
        ├── empty/
        │   ├── db.kdbx
        │   └── password.txt
        ├── single-entry/
        │   ├── db.kdbx
        │   └── password.txt
        ├── groups-and-entries/
        │   ├── db.kdbx
        │   └── password.txt
        ├── with-history/
        │   ├── db.kdbx
        │   └── password.txt
        ├── with-attachments/
        │   ├── db.kdbx
        │   ├── password.txt
        │   ├── source-text.txt           # the file you'll attach
        │   └── source-icon.png           # the file you'll attach
        ├── with-custom-icons/
        │   ├── db.kdbx
        │   ├── password.txt
        │   └── source-icon.png           # icon you'll import
        ├── with-custom-data/
        │   ├── db.kdbx
        │   └── password.txt
        ├── with-tags-and-expiry/
        │   ├── db.kdbx
        │   └── password.txt
        ├── with-autotype/
        │   ├── db.kdbx
        │   └── password.txt
        ├── with-keyfile/
        │   ├── db.kdbx
        │   ├── keyfile.bin
        │   └── password.txt
        ├── kdbx41-features/
        │   ├── db.kdbx
        │   └── password.txt
        ├── kdbx40-legacy/                 # READ-ONLY test
        │   ├── db.kdbx
        │   └── password.txt
        └── kdbx3-legacy/                  # READ-ONLY test
            ├── db.kdbx
            └── password.txt
```
